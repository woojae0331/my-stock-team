# report-pptx

종목 리서치 마크다운(`reports/{종목명}.md`)을 읽어 **KB 옐로우 톤의 디자인된 PPTX**(`reports/{종목명}.pptx`)로 내보냅니다.

동작은 2단계입니다:
1. **(Claude)** 리포트 .md를 읽고 슬라이드별 구조화 **spec(JSON)** 으로 정리합니다 — 가드레일은 이 단계에서 적용합니다.
2. **(렌더러)** `build_pptx.py`가 spec을 받아 디자인된 .pptx로 그립니다.

## 실행 절차

1. 대상 종목의 리포트 파일 `reports/{종목명}.md`를 읽습니다. (없으면 사용자에게 경로/종목명을 확인)
2. 의존성 확인: `python -c "import pptx"` — 실패하면 `pip install python-pptx` 안내 후 설치.
3. 아래 **spec 스키마**에 맞춰 내용을 채운 JSON을 `reports/{종목명}.spec.json`에 저장합니다.
4. 렌더러 실행:
   ```
   python .claude/skills/report-pptx/build_pptx.py --spec "reports/{종목명}.spec.json" --out "reports/{종목명}.pptx"
   ```
5. 생성 성공 메시지를 확인하고 사용자에게 출력 경로를 알립니다.

## 슬라이드 순서 (항상 고정)

표지(종목명·작성일) → 종목 개요 → 재무 요약(표+최근 3개년) → 가격/추세 → 뉴스·심리 → 리스크 → 한 줄 종합

이 순서·구성은 렌더러에 하드코딩되어 있습니다. spec의 키만 채우면 됩니다.

## spec 스키마

```json
{
  "cover":   { "stock_name": "삼성전자", "stock_code": "005930", "date": "2026-06-11", "subtitle": "종목 리서치 리포트" },
  "overview":{
    "kpis": [ { "label": "FY2025 매출", "value": "333.6조", "source": "DART, 2025-12-31" } ],
    "bullets": ["사업 개요 한 줄 (출처/기준일)", "..."]
  },
  "financials": {
    "table": {
      "columns": ["구분", "FY2023", "FY2024", "FY2025"],
      "rows": [["매출", "258.9조", "300.9조", "333.6조"], ["영업이익", "6.6조", "32.7조", "43.6조"]],
      "source": "DART 사업보고서·Naver Finance, 연결기준, 기준일 각 회계연도말"
    },
    "bullets": ["밸류에이션·해석 한 줄 (출처/기준일)"]
  },
  "price_trend": {
    "table": { "columns": ["항목","값"], "rows": [["3개월 등락률","+57.6%"]], "source": "FinanceDataReader, 기준일 2026-06-11" },
    "bullets": ["추세 코멘트 (출처/기준일)"],
    "image": "reports/{종목명}_chart.png"
  },
  "news": { "bullets": ["핵심 이슈 — (매체, 날짜)"], "sentiment": "긍정 (단, 과열·변동성 경계 병존)" },
  "risk": { "bullets": ["1. 리스크명 — 내용 (근거/출처·기준일)"] },
  "summary": { "text": "한 줄 종합 — 판단 근거까지만, 매수·매도 단정 없이" }
}
```

- 모든 키는 선택적입니다(없으면 해당 슬라이드는 빈 영역). 가능한 한 모두 채웁니다.
- `overview.kpis`(선택, 최대 4개): 핵심 지표를 상단 메트릭 카드로 강조합니다. `{label, value, source}`.
- `financials.table`의 값이 숫자로 파싱되면(예: "333.6조") 렌더러가 **네이티브 막대 차트를 자동 생성**해 표 우측에 배치합니다. %가 섞인 행은 차트에서 제외됩니다. 따라서 재무 표는 단위가 일관된 **최근 3개년** 손익 행(매출·영업이익·순이익 등)으로 구성하는 것이 좋습니다.
- `price_trend.image`는 차트 이미지(PNG)가 있을 때만. 없으면 `table`/`bullets`로 대체합니다.

## 가드레일 (spec 작성 시 반드시 준수)

- **모든 수치 옆에 (출처/기준일)** 을 적습니다. 표는 `source` 필드로, 불릿은 문장 끝에 `(출처: …, 기준일: …)`.
- **출처·기준일을 확인할 수 없는 수치는 spec에 넣지 않습니다.**
- **매수·매도·목표가 단정 표현 금지** — "사야 합니다", "목표가 ○○원" 등 금지. 판단 근거까지만 서술하고 "~입니다" 체를 씁니다.
- **표가 잘리지 않게** — 렌더러가 행수 상한(헤더 포함 11행)을 넘으면 자동으로 잘라내고 "외 N행 생략"을 표시하지만, 애초에 핵심 행만 추려 표를 간결히 구성합니다.

## 디자인 (렌더러 고정값 — 바꾸지 말 것)

- 포인트색 **KB 옐로우 `#FFBC00`**, 본문은 그레이·화이트의 차분한 금융 리포트 톤.
- 한글 폰트는 **맑은 고딕** 하나로 고정(글자 깨짐 방지).
- 16:9, 각 슬라이드 하단에 학습·참고용 면책 문구 자동 삽입.
